# claude-review-pdca 公開向け説明メモ

作成日: 2026-04-30

## 一言でいうと

過去のコードレビュー指摘を忘れず、次回の実装時に関連する指摘だけを再提示する、AIコードレビューのPDCA基盤です。

## クライアント向けの説明粒度

### 1. 何の課題を解くものか

AIエージェントや人間がコードレビューをしても、指摘がその場限りで終わると、同じ種類のミスが別ファイル・別セッションで繰り返されます。

`claude-review-pdca` は、レビュー指摘を単なるコメントで終わらせず、次の実装時に参照できる「再利用可能な改善知」に変換します。

### 2. 具体的に何をしているか

- レビューで見つかった指摘を SQLite に保存する
- 次に同じ repo / 近いファイルを編集するとき、関連する未解決指摘だけを注入する
- 毎回すべてを表示せず、対象ファイル・カテゴリ・類似度で絞る
- レビュー実行は毎編集ではなく、一定回数の編集後やセッション終了時にまとめる
- AIが自分に都合よく「これは解決済み」と判断しないよう、dismiss は人間承認に寄せる
- 繰り返し出る指摘は、必要に応じて repo ルールへ昇格する

### 3. 普通のレビュー bot との違い

一般的なレビュー bot は「そのPR・その差分にコメントする」ことが主目的です。

この repo は、レビュー結果を次回以降の実装文脈へ戻すことが主目的です。つまり、単発レビューではなく、AIエージェントの作業品質を継続的に上げるための記憶・再注入レイヤーです。

### 4. 使う場面

- Claude Code / Codex など複数のAIコーディング環境を使う repo
- 同じ種類のレビュー指摘が繰り返される repo
- レビューコメントを rules / AGENTS / CLAUDE.md へ反映する判断を人間が管理したい repo
- その場のレビューだけでなく、次回作業時のミス予防まで含めたい repo

### 5. 動作イメージ

1. 実装中またはレビュー後に指摘が発生する
2. 指摘が repo / ファイル / カテゴリ / 重要度つきで保存される
3. 次回、その repo や関連ファイルを編集するときに、未解決の関連指摘だけが提示される
4. 修正済み・誤検知・保留などの判断は人間承認を前提に扱う
5. 繰り返し発生する指摘は、HITLで repo ルール化を検討する

## 30秒説明

このプロジェクトは、AIコードレビューの指摘をその場限りにしないためのPDCA基盤です。レビューで出た問題をSQLiteに蓄積し、次回同じrepoや関連ファイルを編集するときに必要な指摘だけを再提示します。AIが勝手に指摘を無効化しないよう、人間承認ベースのdismiss方針を持ち、繰り返し出る問題はrepoルールへ昇格できます。単発レビューbotではなく、AI開発の品質改善を継続運用するための仕組みです。
